FAQ

  1. How do I get help or report bugs?


    2. Questions, bug reports, suggestions and ideas are always welcome by email to Saul Youssef.

  2. Is Pacman 3 compatible with previous versions of Pacman?


    3. Yes, all the existing Pacman 2 caches will work as they are. You cannot, however, add Pacman 3 packages to an existing Pacman 2 installation. Instead, you should re-install with Pacman 3.

  3. Where is the list of symbolic cache names? Can I modify it?


    4. Do % pacman -registry to see the default list of symbolic cache names. Authors of Pacman packages can define their own names which are added to your registry when you install the software. See the documentation for the register atom of the Pacman language for more details.

  4. Does Pacman modify files outside of the installation directory tree?


    5. Packages usually keep to the good practice of staying within the directory tree that Pacman creates for them. In general, however, a package may modify files outside of it's directory tree. Because of this, you should always use % pacman -remove to remove packages rather than doing % rm -r -f .

  5. Am I allowed to modify and re-distribute Pacman?


    6. Not without explicit permission from Saul Youssef. See % pacman -license for the details.
